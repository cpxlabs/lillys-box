### 1.2 Food System Freezing

**Status:** :red_circle: Critical
**Priority:** High

#### Current Analysis

Reviewing `src/screens/FeedScene.tsx`, the food system uses:
- `animationState` to control UI interactions
- `handleFeed` function with nested `setTimeout` calls
- `useNavigationList` hook for food selection

#### Potential Issues Identified

1. **Animation State Lock:** If animation doesn't complete, buttons stay disabled forever
   - Line 118: `disabled={animationState !== 'idle' || pet.hunger >= 100}`

2. **Nested setTimeout Risk:** Lines 67-78 have nested timeouts without cleanup
   ```typescript
   setTimeout(() => {
     setAnimationState('happy');
     setTimeout(() => {
       setAnimationState('idle');  // If this never runs, UI stays locked
       setMessage('');
     }, ANIMATION_DURATION.MEDIUM);
   }, ANIMATION_DURATION.MEDIUM);
   ```

3. **No Error Boundary:** If `feed()` throws or `triggerReward` fails, state remains locked

#### Implementation Steps

**File:** `src/screens/FeedScene.tsx`

1. Add cleanup refs for timeouts
2. Add try-catch wrapper around feed logic
3. Add useEffect cleanup to reset state on unmount

```typescript
// Add refs for timeout cleanup
const animationTimeoutRef1 = useRef<NodeJS.Timeout | null>(null);
const animationTimeoutRef2 = useRef<NodeJS.Timeout | null>(null);

// Cleanup on unmount
useEffect(() => {
  return () => {
    if (animationTimeoutRef1.current) clearTimeout(animationTimeoutRef1.current);
    if (animationTimeoutRef2.current) clearTimeout(animationTimeoutRef2.current);
    setAnimationState('idle');
  };
}, []);

const handleFeed = (food: typeof FOODS[0]) => {
  try {
    setAnimationState('eating');
    setMessage(t('feed.eating', { name: pet.name, food: t(food.nameKey) }));

    feed(food.value);

    const moneyEarned = AdsConfig.rewards.feedReward;

    animationTimeoutRef1.current = setTimeout(() => {
      setAnimationState('happy');
      setMessage(t('feed.loved', { name: pet.name }));

      animationTimeoutRef2.current = setTimeout(() => {
        setAnimationState('idle');
        setMessage('');
        triggerReward(moneyEarned);
      }, ANIMATION_DURATION.MEDIUM);
    }, ANIMATION_DURATION.MEDIUM);
  } catch (error) {
    logger.error('Feed error:', error);
    setAnimationState('idle');
    setMessage('');
  }
};
```

};

## 4. Performance/Balance

### 4.1 Energy Decaying Too Fast

**Status:** :warning: Balance Issue
**Priority:** Medium

#### Current State

**File:** `src/config/gameBalance.ts` (lines 9-17)

```typescript
decay: {
  hunger: 1.0,        // -1% per minute
  hygiene: 1.0,       // -1% per minute
  energyDay: 0.5,     // -0.5% per minute (6 AM - 10 PM)
  energyNight: 0.2,   // -0.2% per minute (10 PM - 6 AM)
  // ...
},
```

**File:** `src/utils/petStats.ts` (lines 75-81)
```typescript
export const getEnergyDecayRate = (): number => {
  const hour = new Date().getHours();
  const { dayStartHour, nightStartHour } = GAME_BALANCE.time;

  if (hour >= dayStartHour && hour < nightStartHour) {
    return GAME_BALANCE.decay.energyDay;
  }
  return GAME_BALANCE.decay.energyNight;
};
```

#### Analysis

Current rates:
- Day: -0.5% per minute = -30% per hour = 100% to 0% in ~3.3 hours
- Night: -0.2% per minute = -12% per hour = 100% to 0% in ~8.3 hours

#### Recommended Changes

Reduce decay rates by ~50%:

```typescript
decay: {
  hunger: 0.5,        // Changed from 1.0 - slower hunger decay
  hygiene: 0.5,       // Changed from 1.0 - slower hygiene decay
  energyDay: 0.25,    // Changed from 0.5 - much slower energy decay
  energyNight: 0.1,   // Changed from 0.2 - even slower at night
  // ...
},
```

New rates:
- Day: -0.25% per minute = -15% per hour = 100% to 0% in ~6.6 hours
- Night: -0.1% per minute = -6% per hour = 100% to 0% in ~16.6 hours
